# KubeDeck 2.5.2 — план отображения до 2000 Pods и поиска namespace

Статус: реализация и автоматический regression gate завершены; ожидается ручной smoke.

## Цель

Патч решает две небольшие задачи:

1. Расширить максимальный размер страницы resource-таблицы с `1000` до `2000`, чтобы пользователь мог одновременно видеть до 2000 Pods без перехода между страницами.
2. При поиске в Namespace Selector всегда показывать уже выбранные namespace вместе с найденными, чтобы их можно было снять без очистки поискового запроса.

Это изменение касается только отображения уже загруженных строк. Backend сейчас возвращает полный список ресурсов, поэтому новый API-запрос, параметр `limit`, cache или отдельная загрузка не нужны.

## Подтверждённый baseline

- Размеры страницы задаются общим списком `PAGE_SIZE_OPTIONS` в `useResourceTableState.ts`.
- Текущие варианты: `50`, `100`, `200`, `500`, `1000`.
- Максимум `1000`; таблица показывает остальные Pods на следующих страницах.
- `renderedRows` уже ограничивается обычным `slice(pageStart, pageStart + pageSize)`.
- Все resource-таблицы используют один selector размера страницы.
- Namespace Selector сейчас фильтрует список только по совпадению с поисковой строкой.
- Выбранный namespace исчезает из списка, если не совпадает с новым запросом, поэтому для замены выбора приходится очищать поиск, находить старый namespace и снимать его отдельно.

## Границы 2.5.2

В патч входят только:

- вариант `2000` в существующем selector размера страницы;
- сохранение выбранных namespace в отфильтрованном списке;
- целевой renderer contract;
- проверка большой Pods-таблицы и sticky header;
- version/release metadata 2.5.2 после реализации.

Не входят:

- изменение default page size `200`;
- автоматический выбор `2000`;
- виртуализация строк;
- новый backend endpoint или pagination API;
- отдельная настройка только для Pods;
- отдельный режим «заменить namespace», дополнительная кнопка или новый selector;
- изменение правил multi-select, `all` и cluster-scoped namespace;
- новые зависимости.

Поскольку список размеров страницы общий, вариант `2000` будет доступен и другим resource-таблицам. Отдельная ветка для Pods не нужна.

## Минимальная реализация

- В `PAGE_SIZE_OPTIONS` добавить `2000` после `1000`.
- Не менять `DEFAULT_PAGE_SIZE = 200`.
- Не добавлять специальное состояние, config field или миграцию UI preferences.
- Не менять получение Pods, metrics, watch, polling и фильтрацию.

Ожидаемый production diff: одна константа в одном файле.

## Namespace Selector — минимальная реализация

- При непустом поиске включать namespace в `filteredNamespaces`, если он совпадает с запросом **или уже выбран**.
- Сохранять исходный порядок `namespaces`; не создавать второй список или отдельную секцию Selected.
- Не дублировать namespace, который одновременно выбран и совпадает с запросом.
- Не менять постоянно видимый пункт `All namespaces`.
- После снятия галочки с выбранного namespace, не совпадающего с запросом, он сразу исчезает из результатов.
- Если найденный namespace выбран, оба выбранных значения остаются видимыми до изменения запроса.

Ожидаемый production diff: одно условие в существующем фильтре.
