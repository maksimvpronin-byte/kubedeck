# План полировки KubeDeck после версии 2.1.0

## Цель

Устранить найденные риски в инструментах сборки, убрать предупреждения, упростить поддержку крупных модулей и завершить работу с чистым Git-репозиторием.

План выполняется строго сверху вниз. После завершения пункта его чекбокс меняется с `[ ]` на `[x]`, а под пунктом кратко записываются результат и выполненные проверки.

## Исходное состояние

- Версия приложения синхронизирована: `2.1.0`.
- Electron: `43.1.0`.
- `npm run verify` проходит полностью: 9 renderer-тестов и 69 gateway-тестов.
- `npm audit`: 6 уязвимостей уровня high и 1 уровня moderate в инструментах разработки и сборки.
- Сборка Vite предупреждает об устаревающем CJS Node API.
- `main` опережает `origin/main` на четыре коммита.
- Удаление выполненных планов и обновление regression checklist ещё не закоммичены.

## Правила выполнения

- Не объединять обновление нескольких major-зависимостей в один шаг.
- После каждого изменения запускать указанные проверки.
- Не применять `npm audit fix --force`.
- Не менять рабочую бизнес-логику ради обновления инструментов.
- Если проверка не проходит, не отмечать шаг завершённым; записать ошибку под соответствующим пунктом.
- Перед каждым коммитом проверять `git diff` и не включать сборочные артефакты, секреты и локальные настройки.
