# KubeDeck 1.0.4 Release Notes

## New features

### Node operations

KubeDeck 1.0.4 adds node maintenance actions directly from the Nodes view:

- Bulk `cordon` for selected nodes.
- Bulk `uncordon` for selected nodes.
- Bulk `drain` for selected nodes.

Drain uses the fixed command form:

```powershell
kubectl drain <node> --ignore-daemonsets --delete-emptydir-data --timeout=300s
```

Before drain confirmation, KubeDeck shows a preview of affected pods with namespace and node information.

### In-app confirmation modals

Node actions now use KubeDeck-styled confirmation modals instead of native browser/Windows confirmation dialogs.

### Node SSH

KubeDeck 1.0.4 adds an SSH tab to the Node drawer.

Supported connection options:

- InternalIP as the default node host.
- Username and port.
- SSH agent / default key authentication.
- Username/password authentication.
- Private key authentication.
- Optional jump host.

Secrets are not persisted.

### SSH defaults

Settings now include non-secret SSH defaults:

- default username
- default port
- default auth method
- jump host usage
- jump host
- jump port
- jump username
- jump auth method

KubeDeck does not save SSH passwords, passphrases, or private key paths as defaults.

## Deferred

Helm chart management was removed from 1.0.4 scope and is deferred to a later release.

## Validation checklist

Before tagging `1.0.4`, verify:

- Node list still loads.
- Bulk cordon works.
- Bulk uncordon works.
- Drain modal shows affected pods.
- Drain can be cancelled safely.
- Drain works on a safe test node.
- SSH tab appears in Node drawer.
- SSH defaults are passed from Settings to Node drawer.
- Passwords/passphrases are not saved in `%APPDATA%\KubeDeck\config.json`.
- Existing pod actions still work.
- Existing deployment scale still works.
- Portable Windows build starts successfully.
