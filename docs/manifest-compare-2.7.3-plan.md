# KubeDeck 2.7.3 — исправление Compare и Secret confirmation

Статус: реализация и автоматические проверки завершены; требуется ручной smoke на macOS и Windows.

## Проблемы

1. Длинные YAML выходят за доступную высоту окна Compare, но вертикальная прокрутка колонок фактически недоступна.
2. Переключатель `Clean / Raw` отображается как нативная системная кнопка и визуально не соответствует KubeDeck.
3. Сохранение изменённого Secret вызывает нативный `window.confirm`, который игнорирует тему и стили KubeDeck.
4. После Reveal требуется лишнее нажатие `Edit`, а textarea использует неподходящее системное оформление.
5. Compare не выделяет одинаковые и отличающиеся строки, поэтому визуальное сравнение длинных YAML остаётся ручным.

## Границы патча

- Исправить только layout и оформление уже существующего Compare.
- Упростить существующий Secret editor: Reveal сразу открывает редактируемое поле, backend update не менять.
- Не менять загрузку YAML, очистку manifest, список совместимых вкладок и active resource tab.
- Не добавлять зависимости и собственные scroll-компоненты: достаточно корректного CSS overflow и существующих button tokens.

## 1. Двойная проверка перед изменением

### Первая проверка — структура layout

- [ ] Воспроизвести отсутствие прокрутки с двумя YAML, превышающими высоту viewport.
- [ ] Проверить цепочку ограничений высоты: `.manifest-compare` → `.manifest-compare-grid` → колонки → `pre`.
- [ ] Подтвердить, что проблема вызвана отсутствующим `min-height: 0`/ограничением overflow у grid-элементов, а не загрузкой или обрезанием YAML.
- [ ] Проверить отдельно вертикальное и горизонтальное переполнение длинных строк.

### Вторая проверка — общий стиль controls

- [ ] Подтвердить, что `Clean / Raw` сейчас является `<button>` без общего класса KubeDeck.
- [ ] Сравнить его с существующим `secondary-btn` в resource toolbar и пагинации.
- [ ] Проверить close button: он уже использует общий `.icon-button` и не требует отдельного изменения.
- [ ] Подтвердить, что системное окно на скриншоте создаётся единственным `window.confirm` в `SecretTab`.
- [ ] Сверить структуру с существующими confirmation modal и общими `.modal-actions`, `.primary`, `.secondary`.

Решение принимается только если обе проверки подтверждают одну и ту же локальную причину.

## 2. Исправить прокрутку Compare

- [ ] Ограничить внутреннюю grid-область доступной высотой modal.
- [ ] Добавить `min-height: 0` всем промежуточным grid-контейнерам, которые могут препятствовать сжатию `pre`.
- [ ] Оставить отдельную вертикальную прокрутку для левого и правого manifest.
- [ ] Сохранить горизонтальную прокрутку для длинных YAML-строк без переноса текста.
- [ ] Не допустить прокрутки всего фонового приложения при наведении на manifest.
- [ ] На узком viewport перевести колонки в вертикальный layout либо сохранить читаемую минимальную ширину с контролируемой прокруткой.
- [ ] Заголовки обеих сторон должны оставаться видимыми над прокручиваемым содержимым.
- [ ] Скроллбар не должен перекрывать последнюю строку YAML.

## 3. Привести Clean / Raw к стилю KubeDeck

- [ ] Использовать существующий глобальный класс `icon-text`; отдельную палитру не создавать.
- [ ] Добавить `type="button"`, чтобы control никогда не выполнял случайный submit.
- [ ] Сохранить текущую семантику: надпись показывает активный режим `Clean` или `Raw`.
- [ ] Сохранить общие hover, focus-visible, disabled, border, background и typography tokens.
- [ ] Проверить оформление во всех доступных темах и на Windows/macOS.
- [ ] Не использовать `secondary-btn`: он ограничен `.resource-table-panel` и не стилизует modal.

### Подсветка различий

- [ ] Использовать line diff из прямой renderer dependency `diff`.
- [ ] Выровнять соответствующие строки двух manifest в общей модели rows.
- [ ] Отмечать одинаковые строки нейтральным `=`, изменённые `~` и warning-tone, добавленные `+` и success-tone, удалённые `−` и danger-tone.
- [ ] Показывать номера строк каждой стороны и сохранять горизонтальную/вертикальную прокрутку.
- [ ] Добавить компактную легенду Same / Changed / Added / Removed на semantic tokens текущей темы.
- [ ] Не добавлять Apply, Merge или собственный diff-алгоритм.

## 4. Заменить нативное Secret confirmation

- [ ] Удалить `window.confirm` из `SecretTab`.
- [ ] Удалить отдельную кнопку и иконку `Edit`.
- [ ] После успешного Reveal текстового значения сразу создать in-memory draft и показать редактируемую textarea.
- [ ] Binary-like и immutable Secret оставить только для просмотра: textarea и Save для них не показывать.
- [ ] Для текстового значения всегда показывать явные Save и Cancel; никакого автоматического сохранения при вводе или Hide.
- [ ] Cancel возвращает раскрытое исходное значение без повторного запроса, Save открывает confirmation.
- [ ] Hide очищает draft этого ключа; auto-hide также не должен сохранять введённые изменения.
- [ ] Оставить редактируемым только один key одновременно; Reveal другого key переводит edit-state на него без отправки предыдущего draft.
- [ ] Оформить textarea только существующими tokens: `--code-bg`/`--surface`, `--text`, `--border`, `--focus-ring`, штатные radius и monospace font.
- [ ] Убрать нативный светлый background, системный border и несогласованный focus outline во всех темах.
- [ ] Сохранить вертикальный resize textarea в пределах карточки и корректное отображение многострочных значений.
- [ ] По Save открывать внутренний modal KubeDeck и хранить pending confirmation только в локальном состоянии Secret tab.
- [ ] В заголовке и тексте показывать cluster, namespace, Secret name и key; decoded и base64 значения никогда не выводить.
- [ ] Использовать существующие modal backdrop/panel/header/actions и кнопки `.secondary`/`.primary`, без новой палитры.
- [ ] Cancel закрывает только confirmation и сохраняет draft для продолжения редактирования.
- [ ] Confirm запускает существующий `updateSecret`; во время запроса повторный Confirm заблокирован.
- [ ] Escape и кнопка закрытия ведут себя как Cancel и не очищают draft.
- [ ] После успеха сохранить текущий lifecycle: очистить edit draft, скрыть значение и перезагрузить metadata.
- [ ] Ошибка остаётся в штатном `ErrorPanel`, modal закрывается либо остаётся открытым согласно существующей политике async actions.
