# KubeDeck 2.7.1 — восстановление вкладок resource drawer

Статус: исправление реализовано; автоматические проверки пройдены, ручной smoke ожидает подтверждения.

## Проблема

После 2.7.0 внутренние вкладки resource drawer открываются, но их содержимое может быть пустым либо визуально обрезанным. На воспроизведении затронуты как минимум YAML и Describe; до исправления считаем потенциально затронутыми Summary, LLM, Related, Events и Logs.

## Цель патча

- Вернуть каждой внутренней вкладке полную доступную высоту и собственный scroll.
- Не терять загруженные данные при переключении сохранённых resource tabs.
- Не нарушить нижнюю панель терминалов, адаптивные resource tabs и namespace selector.
- Не добавлять зависимости и не переделывать drawer целиком.

## 1. Дважды подтвердить причину

### Проверка данных

- [ ] Воспроизвести проблему на двух Pod и последовательно открыть Summary → YAML → Describe → Events → Logs.
- [ ] Проверить ответы `resourceText`, Events и Logs: запрос относится к активному `cluster/resource/namespace/name/uid`, ответ не пустой и не устаревший.
- [x] Проверить lifecycle `PodDrawer`: смена внутренней вкладки не очищает snapshot, а поздний ответ предыдущего ресурса не заменяет текущий.
- [ ] Повторить сценарий после переключения между двумя сохранёнными resource tabs.

### Проверка layout

- [ ] Снять фактические размеры `.content`, `.content-upper`, `.resource-workspace`, `.drawer`, `.drawer-content-fill` и активного viewer/editor.
- [ ] Проверить сценарии без терминала, с раскрытой нижней панелью и со свёрнутой нижней панелью.
- [x] Убедиться, что каждый grid/flex ancestor имеет корректные `min-height: 0`, доступную высоту и не обрезает дочерний scroll-контейнер.
- [ ] Повторить проверку на минимальной и максимальной ширине drawer, а также при нескольких resource tabs.

Решение об изменении принимается только после совпадения результата обеих проверок. Если данные присутствуют, исправляется layout; если геометрия корректна, исправляется lifecycle/request generation. Оба слоя одновременно не меняются без отдельного подтверждения двух дефектов.

Подтверждённая причина: без сохранённых resource tabs единственный `PodDrawer` автоматически занимал первую `auto`-строку двухстрочного `.resource-workspace`, а основная `minmax(0, 1fr)`-строка оставалась пустой. Lifecycle/request generation корректны и не изменялись.
